# State Machine-Based LineBot Development

## Project Overview

Developed a LineBot integrated with a State Machine to control user states and workflow. This approach effectively manages complex conversation flows and user interactions.

- **State Management**: Utilized a state machine model to track and manage the current state of users in conversations.

- **Workflow Control**: Guided conversation flows to different branches and functionalities based on user input and current state.

- **Flexible Interaction**: Implemented complex dialogue logic through state transitions, providing a more natural and personalized user experience.

## Technologies Used

- **LINE Messaging API**: For interacting with the LINE platform, receiving and sending messages.
- **TypeScript**: As the primary development language, used for implementing state machine logic and business processing.
- **Finite State Machine (FSM)**: For designing and implementing state transitions in conversation flows.

## Challenges and Learnings

The main challenge was designing a state machine structure that could handle complex conversation flows while maintaining code simplicity and maintainability. This required careful planning of state transition logic and consideration of various possible user inputs. Another challenge was ensuring the state machine could flexibly respond to unexpected user behaviors while maintaining conversation coherence and naturalness.

## Outcome

The LineBot successfully implemented user state management and workflow control based on a state machine. It can flexibly guide the conversation process according to user input and current state, providing a personalized service experience. This project demonstrates how to apply state machine concepts to chatbot development, providing an effective framework for creating more intelligent and interactive dialogue systems.

# 狀態機的 LineBot 開發

## 項目概述

一個結合狀態機（State Machine）的 LineBot，用於控制用戶狀態和工作流程。這種方法能夠有效管理複雜的對話流程和用戶交互。

- **狀態管理**：利用狀態機模型來追蹤和管理用戶在對話中的當前狀態。

- **工作流程控制**：根據用戶的輸入和當前狀態，引導對話流向不同的分支和功能。

- **靈活交互**：通過狀態轉換實現複雜的對話邏輯，提供更自然、更個性化的用戶體驗。

## 使用的技術

- **LINE Messaging API**：用於與 LINE 平台交互，接收和發送消息。
- **TypeScript**：作為主要的開發語言，用於實現狀態機邏輯和業務處理。
- **Finite State Machine (FSM)**：用於設計和實現對話流程的狀態轉換。

## 挑戰與學習

主要挑戰在於設計一個既能處理複雜對話流程，又能保持代碼簡潔和可維護性的狀態機結構。需要仔細規劃狀態轉換邏輯，並考慮各種可能的用戶輸入情況。另一個挑戰是確保狀態機能夠靈活應對意外的用戶行為，同時保持對話的連貫性和自然性。

## 成果

該 LineBot 成功實現了基於狀態機的用戶狀態管理和工作流程控制。它能夠根據用戶的輸入和當前狀態，靈活地引導對話進程，提供個性化的服務體驗。這個項目展示了如何將狀態機概念應用於聊天機器人開發，為創建更智能、更有互動性的對話系統提供了一個有效的框架。
